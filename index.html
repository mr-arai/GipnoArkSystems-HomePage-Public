PCのブラウザからデバイスのWebページを開いてトリガーを発行し、同じPC上に入れた顧客提供のPythonコードを起動したいイメージです。PC〜デバイス間はWi-Fi接続です。

GA
ブラウザのSTART操作と、同じPC上のPython処理を同期させたい、という要件ですね。
まずはシンプルな実現案として、次の構成が考えられます。

■ 提案構成 - ブラウザを介さず Python から直接 デバイスIP (192.168.x.x) へアクセス - Python がトリガー発行 + 顧客コードを 同一フロー内で実行
Pythonがそのままデバイスに接続してトリガー発行まで担う形です。これなら開始タイミングを1つの制御フローにまとめられます。
14:07
なるほどです。ちなみに現在はPyGameで実装した計測ソフトをマウスクリックで開始しています。この開始タイミングもアイトラッカーと同期できますか？
14:08
GA
はい、可能です！先ほどの構成であれば既存のPythonコードを活かしつつ、アイトラッカーへのトリガー送信とPyGame側の開始処理を同じスクリプト内で順序よく制御できます。
14:09
研究用途なので、Pythonから計測機器と解析ツールをまとめて扱える形だと助かります。
14:10
GA
であれば、デバイス側にシンプルなAPIを用意する案が相性よいです。

GET /api/trigger?state=on → これだけでトリガーON # Python から1行で呼び出し可能 requests.get("http://192.168.x.x /api/trigger?state=on")
